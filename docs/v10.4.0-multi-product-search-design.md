# v10.4.0 — Çoklu Ürün Arama Paneli Teknik Tasarım

Durum: DEVAM / LOCAL TEST BEKLİYOR

Bu doküman, Melisa Bebe ERP depo terminaline barkod dışında stok kodu, ürün adı ve stok no ile read-only ürün arama eklemek için teknik tasarım notudur.

## Hedef

Depo ve satış ekibi, ürünleri yalnızca barkodla değil aşağıdaki yöntemlerle de bulabilmelidir:

- Barkod
- Stok Kodu
- Ürün Adı / MALINCINSI
- Stok No / IND

## Kapsam

Öncelikli ekran:

- `src/pages/VegaImportPreview.jsx`
- `src/vegaImport.css`

Gerekirse read-only servis:

- `electron/vegaReadOnlyService.cjs`

## Kesin Güvenlik Kuralları

- Sadece read-only SELECT.
- Parametreli sorgu zorunlu.
- TOP 20 limiti korunacak.
- Fail-closed davranış korunacak.
- Ham SQL hatası kullanıcıya gösterilmeyecek.
- `.env.local`, credential, token, secret renderer tarafına taşınmayacak.
- INSERT / UPDATE / DELETE yok.
- ALTER / DROP / TRUNCATE / CREATE yok.
- GRANT / DENY / REVOKE yok.
- import / export / sync yok.
- Stok düşme, sayım kaydetme, işlem kaydı yok.
- Cari / kasa / finans / fatura yok.
- `agent-loop` ve `melisababy-deploy` alanlarına dokunulmayacak.

## Servis Tasarımı

### Mevcut Barkod Araması

Mevcut fonksiyon korunur:

```js
findVegaProductByBarcodeReadOnly(rawBarcode)
```

### Yeni Önerilen Fonksiyon

```js
searchVegaStockCardsReadOnly({ type, query })
```

Dönüş şekli mevcut güvenli servis patternlerine benzemelidir:

```js
{
  status: "success" | "error",
  message: string,
  items: [],
  columns: ["IND", "STOKKODU", "MALINCINSI"],
  rowCount: number,
  metadata: {
    mode: "stock-card-search-read-only",
    tableScope: "dbo.F0102TBLSTOKLAR",
    maxRowsLimit: 20,
    writeEnabled: false,
    importEnabled: false,
    syncEnabled: false,
    fileOutputEnabled: false,
    rendererReceivesCredentials: false,
    rawErrorExposeEnabled: false
  }
}
```

## SQL Kapsamı

Tek tablo:

```sql
FROM dbo.F0102TBLSTOKLAR s
```

Dönecek alanlar:

```sql
SELECT TOP (20)
  s.IND,
  s.STOKKODU,
  s.MALINCINSI
FROM dbo.F0102TBLSTOKLAR s
```

### Stok Kodu

```sql
WHERE s.STOKKODU LIKE @query
```

Parametre:

```js
request.input("query", sql.NVarChar(50), `%${query}%`)
```

### Ürün Adı

```sql
WHERE s.MALINCINSI LIKE @query
```

Parametre:

```js
request.input("query", sql.NVarChar(50), `%${query}%`)
```

### Stok No / IND

```sql
WHERE s.IND = @stockNo
```

Parametre:

```js
request.input("stockNo", sql.Int, Number(query))
```

## Güvenlik Kontrol Fonksiyonu

Yeni sorgu eklenirse `assertSafeStockCardSearchQuery(queryText, limit, type)` benzeri kontrol eklenmelidir.

Kontrol etmesi gerekenler:

- Limit tam olarak 20.
- Yasaklı SQL kelimeleri yok.
- Tablo kapsamı sadece `dbo.F0102TBLSTOKLAR`.
- Parametreli WHERE kullanımı var.
- Beklenmeyen JOIN yok.
- ORDER BY güvenli bir alanla yapılırsa sadece `STOKKODU` veya `IND` kullanılmalı.

## Renderer / Electron Bridge Notu

Eğer preload veya IPC tarafında yeni kanal gerekiyorsa kanal adı net ve read-only olmalı:

```text
vega:readonly-stock-card-search
```

Bu kanal:

- Credential taşımaz.
- SQL text almaz.
- Sadece `type` ve `query` alır.
- Servis tarafı tip ve query validasyonu yapar.
